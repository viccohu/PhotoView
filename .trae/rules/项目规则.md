架构规则
[Architecture Rules]
- 必须使用 MVVM 架构
- View 不允许包含业务逻辑
- 所有 UI 绑定必须通过 ViewModel
- 服务层（ImageService / CacheService）必须独立
- 禁止在 code-behind 中写复杂逻辑

性能规则
[Performance Rules]
- 必须支持 UI 虚拟化（ItemsRepeater + VirtualizingLayout）
- 禁止一次性加载全部图片
- 图片加载必须异步（async/await）
- 小图必须使用缩略图（thumbnail）而不是原图
- 滚动必须保持 60 FPS（避免 UI 线程阻塞）
- 优先使用微软官方成熟api，避免使用自定义实现
- 优先使用微软推荐的控件库，避免使用自定义实现的控件
- 禁止使用第三方库，除非必要
- 必须考虑线程安全
- 必须考虑大量图片加载的性能问题，避免内存泄漏和性能问题

ui规则
- 所有 UI 元素必须符合 Windows UI 规范
