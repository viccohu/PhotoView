0. 总目标（必须遵守）
构建一个基于 WinUI 3 的高性能图片浏览器，
要求：
- 不崩溃（线程安全）
- 支持虚拟化
- 支持大规模图片（1万+）
- 滚动流畅（不卡顿、不闪烁、不乱序）
- 不错用非winui3的属性或方法
1. 线程模型（最重要规则）
规则 1.1：UI 对象必须在 UI 线程创建
禁止在后台线程创建以下对象：
- BitmapImage
- ImageSource
- DependencyObject
规则 1.2：禁止在 async 后直接操作 UI
禁止：
thumbnailImage.Source = await xxx
VisualStateManager.GoToState(...)

规则 2.1：UI 不负责加载图片
禁止：
- 在 Page / View 中加载图片
- 在 ElementPrepared 里操作 Image.Source
规则 3.1：必须防止元素复用错位
所有 async 加载必须具备：
- version 校验 或
- CancellationToken
规则 3.2：禁止遍历 Repeater 所有元素
规则 4.3：限制并发加载
SemaphoreSlim(4~8)
规则 5.1：禁止加载原图作为缩略图
规则 6.1：UI 必须完全绑定驱动
规则 6.2：禁止代码控制 UI 状态
规则 7.1：禁止吞异常
规则 7.2：Win32异常不可忽略
    必须检查：
    - 线程
    - UI对象访问
