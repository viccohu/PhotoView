# 快速切换目录崩溃修复计划

## 问题分析

### 核心问题
快速切换目录时，程序崩溃的原因：
1. **旧的缩略图加载任务没有被取消** - `ImageFileInfo` 内部的 `_thumbnailLoadCts` 没有被取消
2. **Images.Clear() 前没有清理缩略图任务** - 直接清空集合但后台任务还在运行
3. **对象销毁后任务仍在访问** - 旧的 ImageFileInfo 对象可能已被 GC 回收，但加载任务还在继续

### 问题细节
- **MainViewModel.cs:262** - `Images.Clear()` 直接清空，但没有先调用 `ClearThumbnail()`
- **ImageFileInfo.cs** - 每个 ImageFileInfo 有自己的 `_thumbnailLoadCts`，目录切换时没有被取消
- **没有等待任务取消完成** - 立即切换到新目录，旧任务可能还在访问 UI 线程

## 修复方案

### 1. 修改 MainViewModel.LoadImagesAsync 方法
在清空 Images 之前，先调用所有现有 ImageFileInfo 的 ClearThumbnail() 方法

### 2. 确保任务正确取消
- 旧任务在被取消后不会继续执行 UI 更新
- 旧任务在被取消后释放资源

## 修改文件

- `ViewModels/MainViewModel.cs` - 修改 LoadImagesAsync 方法，在清空前清理缩略图任务

## 实施步骤

1. 在 LoadImagesAsync 方法的 Images.Clear() 之前
2. 遍历当前 Images 集合中的所有 ImageFileInfo
3. 调用每个 ImageFileInfo 的 ClearThumbnail() 方法
4. 然后再执行 Images.Clear()
