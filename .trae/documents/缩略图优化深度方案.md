# 缩略图加载深度优化方案

基于 `缩略图优化2.md` 文档中的问题点，结合当前项目实际代码，制定以下优化方案。

---

## 一、问题诊断

### 1. 致命崩溃点：DispatcherQueue 任务堆积与资源竞争

**当前问题：**
- `ImageFileInfo.cs:110-135` 的 `EnsureThumbnailAsync` 方法虽然有 `_loadVersion` 机制，但缺少 `CancellationTokenSource`
- 快速滚动时，大量 `GetThumbnailAsync` 和 `CreateBitmapOnUIThreadAsync` 任务被触发
- UI 线程队列被占满，导致内存溢出或 WinUI 3 内部资源访问冲突

**现有实现：**
```csharp
// ImageFileInfo.cs:110-135
public async Task EnsureThumbnailAsync(ThumbnailSize size)
{
    if (Thumbnail != null || _isLoading)
        return;

    _isLoading = true;
    var version = ++_loadVersion;

    try
    {
        var result = await GetThumbnailAsync(size);

        if (version != _loadVersion)
            return;

        Thumbnail = result;
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"EnsureThumbnailAsync error: {ex}");
    }
    finally
    {
        _isLoading = false;
    }
}
```

## 二、优化方案

### 优先级 0：解决快速滚动崩溃（最高优先级）

#### 2.1 ImageFileInfo.cs - 引入 CancellationTokenSource

**修改点：**
1. 添加 `CancellationTokenSource? _thumbnailLoadCts` 字段
2. 修改 `EnsureThumbnailAsync` 方法，在开始前取消旧任务
3. 修改 `GetThumbnailAsync` 支持 CancellationToken
4. 修改 `CreateBitmapOnUIThreadAsync` 支持 CancellationToken
5. 降低 DispatcherQueue 优先级为 Low

**代码示例：**
```csharp
public class ImageFileInfo : INotifyPropertyChanged
{
    private BitmapImage? _thumbnail;
    private bool _isLoading;
    private int _loadVersion;
    private CancellationTokenSource? _thumbnailLoadCts; // 新增
    
    // ... 现有代码 ...
    
    public async Task EnsureThumbnailAsync(ThumbnailSize size)
    {
        _thumbnailLoadCts?.Cancel();
        _thumbnailLoadCts = new CancellationTokenSource();
        var cancellationToken = _thumbnailLoadCts.Token;
        
        _isLoading = true;
        var version = ++_loadVersion;

        try
        {
            var result = await GetThumbnailAsync(size, cancellationToken);

            if (version != _loadVersion || cancellationToken.IsCancellationRequested)
                return;

            Thumbnail = result;
        }
        catch (OperationCanceledException)
        {
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine($"EnsureThumbnailAsync error: {ex}");
        }
        finally
        {
            _isLoading = false;
        }
    }
    
    public void CancelThumbnailLoad()
    {
        _thumbnailLoadCts?.Cancel();
        _loadVersion++;
    }
    
    private async Task<BitmapImage> GetThumbnailAsync(ThumbnailSize size, CancellationToken cancellationToken)
    {
        try
        {
            var requestedSize = (uint)size;
            var optimalSize = GetOptimalThumbnailSize(requestedSize);

            StorageItemThumbnail thumbnail = await ImageFile.GetThumbnailAsync(
                ThumbnailMode.PicturesView,
                optimalSize,
                ThumbnailOptions.UseCurrentScale).AsTask(cancellationToken);

            if (thumbnail != null && thumbnail.Size > 0)
            {
                var bitmapImage = await CreateBitmapOnUIThreadAsync(thumbnail, cancellationToken);
                thumbnail.Dispose();
                return bitmapImage;
            }

            return new BitmapImage();
        }
        catch (OperationCanceledException)
        {
            throw;
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine($"GetThumbnailAsync error: {ex}");
            return new BitmapImage();
        }
    }
    
    private static async Task<BitmapImage> CreateBitmapOnUIThreadAsync(IRandomAccessStream stream, CancellationToken cancellationToken)
    {
        BitmapImage? bitmap = null;

        var dispatcherQueue = App.MainWindow.DispatcherQueue;
        if (dispatcherQueue == null)
        {
            bitmap = new BitmapImage();
            bitmap.SetSource(stream);
            return bitmap;
        }

        var tcs = new TaskCompletionSource<bool>();
        
        using var registration = cancellationToken.Register(() => tcs.TrySetCanceled());

        dispatcherQueue.TryEnqueue(DispatcherQueuePriority.Low, () =>
        {
            try
            {
                if (cancellationToken.IsCancellationRequested)
                {
                    tcs.TrySetCanceled();
                    return;
                }
                
                bitmap = new BitmapImage();
                bitmap.SetSource(stream);
                tcs.SetResult(true);
            }
            catch (Exception ex)
            {
                tcs.SetException(ex);
            }
        });

        await tcs.Task;
        return bitmap!;
    }
}
```

### 优先级 1：优化加载性能

#### 2.3 MainViewModel.cs - 使用 Task.WhenAll 并行加载

**修改点：**
- 将串行的 `foreach` 改为并行的 `Task.WhenAll`
- 保持 `Task.Run` 以避免阻塞 UI 线程

**代码修改 (MainViewModel.cs:170-191)：**
```csharp
var imageInfos = await System.Threading.Tasks.Task.Run(async () =>
{
    var tasks = new List<Task<ImageFileInfo?>>();
    
    foreach (var file in batch)
    {
        if (cancellationToken.IsCancellationRequested)
            break;
            
        if (IsImageFile(file))
        {
            tasks.Add(LoadImageInfoSafeAsync(file, cancellationToken));
        }
    }
    
    var results = await Task.WhenAll(tasks);
    return results.Where(r => r != null).Cast<ImageFileInfo>().ToList();
}, cancellationToken);

// 新增辅助方法
private static async Task<ImageFileInfo?> LoadImageInfoSafeAsync(StorageFile file, CancellationToken cancellationToken)
{
    try
    {
        var properties = await file.Properties.GetImagePropertiesAsync().AsTask(cancellationToken);
        return new ImageFileInfo(properties, file, file.DisplayName, file.DisplayType);
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"LoadImageInfo error: {ex}");
        return null;
    }
}
```

**修改点：**

private void SelectionService_SelectionChanged(object? sender, Services.SelectionChangedEventArgs e)

        }
    }
}

private void ImageBorder_PointerPressed(object sender, PointerRoutedEventArgs e)

    {

        // 移除 UpdateItemSelectionStates() 调用
        e.Handled = true;
    }

    if (e.Key == VirtualKey.A)
    {
        var isCtrlPressed = InputKeyboardSource.GetKeyStateForCurrentThread(VirtualKey.Control).HasFlag(Windows.UI.Core.CoreVirtualKeyStates.Down);
        if (isCtrlPressed)
        {
            SelectionService.SelectAll(ViewModel.Images.ToList());
            // 移除 UpdateItemSelectionStates() 调用
            e.Handled = true;
        }
    }
    else if (e.Key == VirtualKey.Escape)
    {
        SelectionService.ClearSelection();
        // 移除 UpdateItemSelectionStates() 调用
        e.Handled = true;
    }
}

## 四、关键技术要点

### 1. CancellationTokenSource 的正确使用
- 每个 `ImageFileInfo` 实例独立管理自己的 CTS
- 在 `EnsureThumbnailAsync` 开始前取消旧任务
- 在 `ElementCleared` 时也取消任务
- 注意 `OperationCanceledException` 的处理

### 2. DispatcherQueuePriority.Low
- 确保缩略图加载不会抢占用户交互的优先级
- 滚动操作会优先响应

### 3. Task.WhenAll 的合理使用
- 只在页内并行，保持分页机制
- 控制并行度，避免磁盘 IO 竞争
- 使用辅助方法处理单个任务的异常

### 4. 数据绑定驱动 UI
- 利用 MVVM 模式，减少 code-behind 逻辑
- 通过属性变化通知驱动 UI 更新
- 避免手动遍历所有元素
