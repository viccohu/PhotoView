# 缩略图尺寸切换优化计划

## [ ] 任务 1：分析尺寸切换时的缩略图加载逻辑
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 分析当前尺寸切换时的缩略图加载逻辑
  - 确定为什么切换尺寸后加载慢且不完整
  - 检查 GridView 可见元素的获取方法
- **成功标准**：
  - 理解问题根因
  - 确定解决方案
- **测试要求**：
  - `programmatic` TR-1.1：验证尺寸切换时缩略图被清除但未重新加载
  - `human-judgement` TR-1.2：观察切换尺寸后的加载行为
- **注意**：需要考虑 GridView 虚拟化机制

## [ ] 任务 2：实现尺寸切换时的缩略图重新加载
- **优先级**：P1
- **依赖**：任务 1
- **描述**：
  - 在 ViewModel_ThumbnailSizeChanged 中实现可见元素的缩略图重新加载
  - 确保第一张图片和所有可见元素都能重新加载
  - 优化加载顺序和并发控制
- **成功标准**：
  - 尺寸切换后所有可见元素的缩略图能立即开始加载
  - 加载速度快且完整
  - 不影响滚动性能
- **测试要求**：
  - `programmatic` TR-2.1：验证切换尺寸后可见元素的缩略图重新加载
  - `programmatic` TR-2.2：验证第一张图片能正确加载
  - `human-judgement` TR-2.3：评估加载速度和完整性
- **注意**：需要考虑性能，避免一次性加载过多

## [ ] 任务 3：测试验证
- **优先级**：P2
- **依赖**：任务 2
- **描述**：
  - 测试不同尺寸切换场景：中-小-中
  - 验证滚动时的加载行为
  - 测试大数据量下的表现
- **成功标准**：
  - 所有尺寸切换场景都能正常加载
  - 第一张图片始终能加载
  - 滚动时加载流畅
- **测试要求**：
  - `programmatic` TR-3.1：测试中-小-中切换场景
  - `programmatic` TR-3.2：测试滚动加载
  - `human-judgement` TR-3.3：评估整体用户体验
- **注意**：测试不同数量级的图片（100+）
