## 四、详细问题列表

### 🔴 高优先级问题

#### 1. CollectViewModel 硬编码路径
**位置**: CollectViewModel.cs:23
```csharp
StorageFolder picturesFolder = await StorageFolder.GetFolderFromPathAsync("D:\\test");
```
**问题**: 硬编码路径导致应用不可移植
**建议**: 使用配置服务或用户选择

#### 2. ShellViewModel 事件未取消订阅
**位置**: ShellViewModel.cs:40
```csharp
NavigationService.Navigated += OnNavigated;
```
**问题**: 未实现 IDisposable，事件订阅永不释放
**建议**: 实现 IDisposable 并取消订阅

#### 3. MainPage 事件未取消订阅
**位置**: MainPage.xaml.cs:27-29
```csharp
SelectionService.SelectionChanged += SelectionService_SelectionChanged;
ViewModel.ImagesChanged += ViewModel_ImagesChanged;
ViewModel.ThumbnailSizeChanged += ViewModel_ThumbnailSizeChanged;
```
**问题**: Unloaded 中只调用了 ViewModel.Dispose()，未取消事件订阅
**建议**: 在 MainPage_Unloaded 中取消所有事件订阅

---

### 🟡 中优先级问题

#### 4. ViewModel 直接访问 Storage API
**位置**: MainViewModel.cs:101, 125, 161
**问题**: 违反 MVVM 分层原则
**建议**: 创建 IFileSystemService 封装文件操作

#### 5. Model 包含异步加载逻辑
**位置**: ImageFileInfo.cs:156-233
**问题**: Model 应该是纯数据，不应包含加载逻辑
**建议**: 创建 IThumbnailService 处理缩略图加载

#### 6. 异常被吞掉
**位置**: 多处 catch (Exception) { }
**问题**: 丢失错误信息，调试困难
**建议**: 至少记录日志

#### 7. 资源未释放
**位置**: ImageFileInfo.cs:62-65
```csharp
var thumbnail = await ImageFile.GetThumbnailAsync(...);
// thumbnail 未显式 Dispose
```
**问题**: 可能导致资源泄漏
**建议**: 使用 using 语句

---

### 🟢 低优先级问题

#### 8. 缺少 IFileService 实际使用
**位置**: App.xaml.cs:65
**问题**: 注册了服务但未使用
**建议**: 移除或在 ViewModel 中使用

#### 9. 并发锁竞争
**位置**: ImageFileInfo.cs:161-176
**问题**: lock 内部有异步等待
**建议**: 使用 SemaphoreSlim 替代 lock

## 六、架构改进建议

### 1. 引入服务层
```
ViewModels/
├── MainViewModel.cs
Services/
├── IFileSystemService.cs
├── FileSystemService.cs
├── IThumbnailService.cs
├── ThumbnailService.cs
├── IFolderService.cs
└── FolderService.cs
```

### 2. 改进事件订阅管理
```csharp
// MainPage.xaml.cs
private void MainPage_Unloaded(object sender, RoutedEventArgs e)
{
    SelectionService.SelectionChanged -= SelectionService_SelectionChanged;
    ViewModel.ImagesChanged -= ViewModel_ImagesChanged;
    ViewModel.ThumbnailSizeChanged -= ViewModel_ThumbnailSizeChanged;
    ViewModel.Dispose();
}
```

### 3. 改进异常处理
```csharp
catch (Exception ex)
{
    System.Diagnostics.Debug.WriteLine($"Method error: {ex}");
    // 或使用 ILogger 服务
}
```
