# 先分组后加载优化方案

## 核心思路

```
1. 快速扫描：只获取文件名（不加载图片属性）
2. 立即分组：按文件名建立分组，确定主图片
3. 只加载主图片：主图片优先加载缩略图
4. 后台补充：其他格式信息在后台补充（不加载缩略图）
```

## 优势

| 优势           | 说明                       |
| ------------ | ------------------------ |
| 减少缩略图加载      | 同一张图片只加载一次缩略图            |
| 避免加载 RAW 缩略图 | RAW 缩略图通常很大，优先加载 JPG/PNG |
| 首屏更快         | 不需要等待所有文件扫描完，边扫描边显示      |

## 详细实现

### 核心伪代码

```csharp
// 1. 快速扫描（第一批）
var firstBatch = await result.GetFilesAsync(0, 50);

// 2. 建立文件名字典
var fileNameMap = new Dictionary<string, List<StorageFile>>();
foreach (var file in firstBatch)
{
    var groupName = ImageGroup.GetGroupName(file.Name);
    if (!fileNameMap.ContainsKey(groupName))
        fileNameMap[groupName] = new List<StorageFile>();
    fileNameMap[groupName].Add(file);
}

// 3. 对每个组，只选择主图片加载
foreach (var kvp in fileNameMap)
{
    var groupFiles = kvp.Value;
    var primaryFile = SelectPrimaryFile(groupFiles); // 按优先级选择
    // 只加载 primaryFile 的 ImageInfo 和缩略图
    // 其他文件只记录到 Group 中，暂不加载缩略图
}

// 4. 继续扫描剩余文件...
```

### 关键优化点

1. **使用文件名字典**：快速查找相同文件名，避免 O(n²) 比较
2. **主图片优先**：只对主图片调用 LoadImageInfo 和缩略图加载
3. **缩略图懒加载**：其他格式的文件只保存 StorageFile 引用，不加载缩略图

## 成功标准验证

* **首屏速度**：< 1 秒

* **缩略图加载数**：只加载主图片（约为总文件数的 60-80%）

* **最终分组**：正确折叠

* **无卡顿**：滚动流畅
