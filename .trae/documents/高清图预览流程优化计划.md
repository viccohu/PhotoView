# 高清图预览流程优化计划

## 当前问题

当前流程：
1. 动画阶段：使用缩略图
2. 动画结束后：切换到 ScrollView，使用缩略图
3. 异步加载高清图后：替换为高清图（导致闪烁）

**问题**：用户会看到"缩略图 → 高清图"的切换，产生闪烁

---

## 优化目标

用户期望的流程：
1. **异步加载高清图**（动画期间就开始加载）
2. **动画结束后**：确认高清图已经加载完成
3. **切换到 ScrollView**：直接以高清图进入，Image 使用 Uniform（避免图片闪烁）
4. **渐进式加载**：如果高清图还没加载完，先显示缩略图，等高清图加载完再替换

---

## 解决方案

### 核心思路
- 动画期间启动高清图异步加载
- 动画结束后等待高清图加载完成（带超时）
- 切换到 ScrollView 时直接使用高清图
- 使用 Uniform 模式，无需手动计算缩放

---

## 修改步骤

### 步骤 1：添加高清图加载任务字段
```csharp
private Task<DecodeResult?>? _highResLoadTask;
```

### 步骤 2：修改 `PrepareContent`
- 启动高清图异步加载（不等待）
- 保存 `Task<DecodeResult?>` 供后续使用

### 步骤 3：修改 `LoadHighResolutionImageAsync`
- 改为返回 `Task<DecodeResult?>`
- 不在内部处理 UI 更新

### 步骤 4：修改 `ShowAfterAnimationAsync`
- 动画结束后，等待高清图加载完成（带 2 秒超时）
- 调用 `SwitchToViewerLayerAsync` 并传入高清图结果

### 步骤 5：修改 `SwitchToViewerLayerAsync`
- 如果高清图已加载，直接使用高清图
- 否则使用缩略图
- 使用 Uniform 模式，无需手动计算缩放

### 步骤 6：添加渐进式加载逻辑
- 如果高清图加载超时，先显示缩略图
- 高清图加载完成后，再替换

---

## 预期效果

| 场景 | 行为 |
|-----|------|
| 高清图加载快 | 动画结束后直接显示高清图，无闪烁 |
| 高清图加载慢 | 动画结束后显示缩略图，高清图加载完再替换 |
| 网络或大文件 | 最多等待 2 秒，避免长时间黑屏 |

### 关键优势
- ✅ 避免图片闪烁
- ✅ 用户体验流畅
- ✅ 代码简洁
- ✅ 使用 Uniform 模式，无需手动计算缩放
