# 图片拖动平移替代方案

## 问题分析

当前使用 `ScrollView.ScrollTo()` 实现拖动存在以下问题：
1. **阻力感大** - ScrollTo 方法有内置的动画/惯性
2. **抖动严重** - 即使使用 `ScrollingAnimationMode.Disabled` 仍然不跟手
3. **坐标计算复杂** - 需要处理滚动偏移量和视口坐标的转换

## 替代方案对比

### 方案一：TranslateTransform + Pointer 事件（推荐）

**原理**：直接操作 Image 的 RenderTransform，不经过 ScrollView 的滚动机制

**优点**：
- 完全跟手，无延迟
- 实现简单直观
- 性能最优（直接操作 Transform 属性，独立于 UI 线程动画）
- Microsoft PowerToys 项目采用此方案

**缺点**：
- 需要手动处理边界限制
- 需要手动实现缩放逻辑

**XAML 结构**：
```xml
<Grid x:Name="ImageContainer"
      Background="{ThemeResource SolidBackgroundFillColorBaseBrush}"
      PointerPressed="ImageContainer_PointerPressed"
      PointerMoved="ImageContainer_PointerMoved"
      PointerReleased="ImageContainer_PointerReleased"
      PointerCanceled="ImageContainer_PointerReleased"
      PointerCaptureLost="ImageContainer_PointerReleased"
      PointerWheelChanged="ImageContainer_PointerWheelChanged">
    <Image x:Name="MainImage"
           Stretch="Uniform"
           HorizontalAlignment="Center"
           VerticalAlignment="Center"
           RenderTransformOrigin="0.5,0.5"
           DoubleTapped="MainImage_DoubleTapped">
        <Image.RenderTransform>
            <CompositeTransform x:Name="ImageTransform"/>
        </Image.RenderTransform>
    </Image>
</Grid>
```

**核心代码**：
```csharp
private bool _isPanning = false;
private Point _lastPanPoint;
private double _scale = 1.0;
private double _fitScale = 1.0;
private const double MinScale = 0.1;
private const double MaxScale = 10.0;
private const double ZoomFactor = 1.1;

private void ImageContainer_PointerPressed(object sender, PointerRoutedEventArgs e)
{
    if (e.Pointer.PointerDeviceType == PointerDeviceType.Mouse)
    {
        var properties = e.GetCurrentPoint(ImageContainer).Properties;
        if (properties.IsLeftButtonPressed)
        {
            _isPanning = true;
            _lastPanPoint = e.GetCurrentPoint(ImageContainer).Position;
            ImageContainer.CapturePointer(e.Pointer);
            e.Handled = true;
        }
    }
}

private void ImageContainer_PointerMoved(object sender, PointerRoutedEventArgs e)
{
    if (_isPanning && e.Pointer.PointerDeviceType == PointerDeviceType.Mouse)
    {
        var currentPoint = e.GetCurrentPoint(ImageContainer);
        var dx = currentPoint.Position.X - _lastPanPoint.X;
        var dy = currentPoint.Position.Y - _lastPanPoint.Y;
        
        ImageTransform.TranslateX += dx;
        ImageTransform.TranslateY += dy;
        
        _lastPanPoint = currentPoint.Position;
        e.Handled = true;
    }
}

private void ImageContainer_PointerReleased(object sender, PointerRoutedEventArgs e)
{
    if (_isPanning)
    {
        _isPanning = false;
        ImageContainer.ReleasePointerCapture(e.Pointer);
        e.Handled = true;
    }
}

private void ImageContainer_PointerWheelChanged(object sender, PointerRoutedEventArgs e)
{
    var ctrl = InputKeyboardSource.GetKeyStateForCurrentThread(VirtualKey.Control);
    if (ctrl.HasFlag(CoreVirtualKeyStates.Down))
    {
        var delta = e.GetCurrentPoint(ImageContainer).Properties.MouseWheelDelta;
        var zoomDelta = delta > 0 ? ZoomFactor : 1.0 / ZoomFactor;
        var newScale = _scale * zoomDelta;
        
        if (newScale >= MinScale && newScale <= MaxScale)
        {
            _scale = newScale;
            ImageTransform.ScaleX = _scale;
            ImageTransform.ScaleY = _scale;
        }
        e.Handled = true;
    }
}
```

### 方案二：Manipulation 事件

**原理**：使用系统级手势事件 ManipulationDelta

**优点**：
- 系统级手势处理，支持多点触控
- 内置惯性和速度计算
- 支持同时处理平移、缩放、旋转

**缺点**：
- 需要设置 ManipulationMode
- 事件处理相对复杂
- 对于纯鼠标操作可能过于复杂

**XAML 结构**：
```xml
<Grid x:Name="ImageContainer"
      ManipulationMode="TranslateX,TranslateY,Scale"
      ManipulationStarted="ImageContainer_ManipulationStarted"
      ManipulationDelta="ImageContainer_ManipulationDelta"
      ManipulationCompleted="ImageContainer_ManipulationCompleted">
    <Image x:Name="MainImage" RenderTransformOrigin="0.5,0.5">
        <Image.RenderTransform>
            <CompositeTransform x:Name="ImageTransform"/>
        </Image.RenderTransform>
    </Image>
</Grid>
```

**核心代码**：
```csharp
private void ImageContainer_ManipulationDelta(object sender, ManipulationDeltaRoutedEventArgs e)
{
    ImageTransform.TranslateX += e.Delta.Translation.X;
    ImageTransform.TranslateY += e.Delta.Translation.Y;
    
    if (e.Delta.Scale != 1.0)
    {
        _scale *= e.Delta.Scale;
        ImageTransform.ScaleX = _scale;
        ImageTransform.ScaleY = _scale;
    }
}
```

### 方案三：GestureRecognizer

**原理**：使用 GestureRecognizer 类进行手势识别

**优点**：
- 最专业的手势处理方式
- 支持复杂手势组合
- 可自定义手势设置

**缺点**：
- 实现最复杂
- 需要手动处理 Pointer 事件并传递给 GestureRecognizer
- 对于简单拖动场景可能过度设计

## 推荐方案

**推荐使用方案一：TranslateTransform + Pointer 事件**

理由：
1. **性能最优** - 直接操作 Transform 属性，动画独立于 UI 线程
2. **实现简单** - 代码量少，逻辑清晰
3. **完全跟手** - 没有 ScrollView 的内置延迟和惯性
4. **官方验证** - Microsoft PowerToys 项目采用此方案
5. **符合项目规则** - 不需要复杂的异步操作，线程安全

## 实现步骤

### 步骤 1：修改 XAML
- 移除 ScrollView
- 使用 Grid + Image + CompositeTransform 结构
- 添加 Pointer 事件绑定

### 步骤 2：实现拖动逻辑
- PointerPressed：记录起始位置，捕获指针
- PointerMoved：计算位移，更新 TranslateX/Y
- PointerReleased：释放指针捕获

### 步骤 3：实现缩放逻辑
- PointerWheelChanged：Ctrl+滚轮缩放
- 双击切换 1:1/适应模式

### 步骤 4：实现边界限制
- ClampTranslation() 方法限制图片不超出合理范围

### 步骤 5：处理双击缩放
- 双击切换适应模式和 1:1 模式

## 注意事项

1. **RenderTransformOrigin** 必须设置为 `0.5,0.5`，确保缩放以图片中心为基准
2. **边界限制** 需要根据当前缩放级别动态计算
3. **双击缩放** 需要重置 TranslateX/Y 到中心位置
4. **保持 ConnectedAnimation 兼容** - 动画层和查看层分离架构不变
