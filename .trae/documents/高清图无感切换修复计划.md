# 高清图无感切换修复计划

## 问题分析

### 当前问题
高清图替换后还是会跳变大，原因是：
1. 使用 `ImageScrollView.ViewportWidth/Height` 获取的尺寸可能不准确
2. 解码尺寸计算时机不对
3. 缩放比例计算依赖的尺寸不一致

### 用户建议
使用系统 API 获取显示器分辨率来决定解码尺寸：
- 1080p 屏幕 → 固定解码最长边 1080
- 2K 屏幕 → 固定解码最长边 2560
- 4K 屏幕 → 固定解码最长边 3840

---

## 解决方案

### 核心思路
1. **使用显示器分辨率**决定解码尺寸，而不是视口尺寸
2. **缩略图和高清图使用相同的解码尺寸策略**
3. **确保缩放比例计算一致**

### 具体实现

#### 1. 获取显示器分辨率
使用 `DisplayInformation` API 获取当前显示器分辨率：
```csharp
using Windows.Graphics.Display;

var displayInfo = DisplayInformation.GetForCurrentView();
var screenWidth = displayInfo.ScreenWidthInRawPixels;
var screenHeight = displayInfo.ScreenHeightInRawPixels;
var monitorLongSide = (uint)Math.Max(screenWidth, screenHeight);
```

#### 2. 解码尺寸策略
```csharp
// 根据显示器分辨率决定解码最长边
uint targetLongSide = monitorLongSide switch
{
    <= 1920 => 1920,   // 1080p 及以下
    <= 2560 => 2560,   // 2K
    <= 3840 => 3840,   // 4K
    _ => monitorLongSide  // 更高分辨率
};
```

#### 3. 统一缩放计算
确保缩略图和高清图都使用相同的计算方式：
- 解码尺寸基于显示器分辨率
- 缩放比例基于视口尺寸和图片像素尺寸

---

## 修改步骤

### 步骤 1：添加获取显示器分辨率的方法
```csharp
private uint GetMonitorLongSide()
{
    try
    {
        var displayInfo = DisplayInformation.GetForCurrentView();
        var screenWidth = displayInfo.ScreenWidthInRawPixels;
        var screenHeight = displayInfo.ScreenHeightInRawPixels;
        return (uint)Math.Max(screenWidth, screenHeight);
    }
    catch
    {
        return 1920; // 默认 1080p
    }
}
```

### 步骤 2：修改 LoadHighResolutionImageAsync
- 使用显示器分辨率决定解码尺寸
- 不再依赖 ViewportWidth/Height

### 步骤 3：确保无感切换
- 解码尺寸固定后，缩略图和高清图的比例一致
- 替换时只需更新 Source，不需要重新计算缩放
