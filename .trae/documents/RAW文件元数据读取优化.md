# RAW 文件元数据读取优化方案

## 问题分析

当前使用的 API：
```csharp
file.Properties.GetImagePropertiesAsync()
```

这个 API 在读取 RAW 文件时会抛出 `PROPSYS.dll` 的 `8000FFFF` 错误，因为：
- Windows 属性系统对某些 RAW 格式支持不完整
- 无法正确读取 EXIF/元数据

---

## 解决方案

### 方案 A：使用 Windows Imaging Component (WIC) 读取
**优点**：WIC 是 Windows 原生图片处理 API，对 RAW 支持更好
**缺点**：需要使用 COM 互操作，代码较复杂

### 方案 B：优化异常处理 + 降级策略（推荐）
**优点**：简单稳定，不影响功能
**方案**：
1. 优先尝试 `GetImagePropertiesAsync()`
2. 如果失败，用默认宽高（比如 200x200，1:1）
3. 缩略图加载后，可以根据实际缩略图尺寸重新计算

### 方案 C：使用第三方库（如 Magick.NET）
**优点**：功能强大，支持所有格式
**缺点**：增加依赖，体积增大

---

## 推荐实现方案（方案 B）

| 步骤 | 说明 |
|------|------|
| 1. 优化 `LoadImageInfoSafeAsync` | 异常时返回默认宽高 |
| 2. 简化 `ImageFileInfo` 构造 | 支持快速创建（默认宽高） |
| 3. 首屏显示更快 | 不等待元数据 |

---

## 实现任务

### [ ] 任务 1：优化 LoadImageInfoSafeAsync - 异常时返回默认值
- **优先级**：P0
- **描述**：
  - 捕获异常，返回默认宽高的 ImageFileInfo
  - 不返回 null，确保加载流程不中断
- **成功标准**：RAW 文件不会导致加载中断

### [ ] 任务 2：简化 ImageFileInfo - 添加快速构造函数

- **描述**：

  - 或者在现有构造函数中处理 0 宽高
- **成功标准**：可以快速创建占位对象
