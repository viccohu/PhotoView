# 缩放坐标问题分析与修复方案

## 问题描述

当前项目鼠标缩放时坐标不对，缩放后鼠标位置与图片上的点不匹配。

## 根本原因分析

### 坐标系统差异对比

#### 参考项目 (PhysicsImageViewer.cs) 的坐标系统

```csharp
// 1. 鼠标位置转换为相对于容器中心的坐标
var containerCenterX = _container.ActualWidth / 2;
var containerCenterY = _container.ActualHeight / 2;
var mouseRelativeToCenterX = mouse.X - containerCenterX;
var mouseRelativeToCenterY = mouse.Y - containerCenterY;

// 2. 计算缩放锚点（图片坐标系）
_zoomAnchorImgX = (mouseRelativeToCenterX - _translateX) / _zoomScale;
_zoomAnchorImgY = (mouseRelativeToCenterY - _translateY) / _zoomScale;
_zoomAnchorScreenX = mouseRelativeToCenterX;  // 保存相对于中心的坐标
_zoomAnchorScreenY = mouseRelativeToCenterY;

// 3. 应用缩放时直接计算新的位移
if (_hasZoomAnchor)
{
    _translateX = _zoomAnchorScreenX - _zoomAnchorImgX * _zoomScale;
    _translateY = _zoomAnchorScreenY - _zoomAnchorImgY * _zoomScale;
}
```

**关键点**：
- `_translateX/Y` 是相对于**容器中心**的偏移
- 缩放锚点坐标也是相对于**容器中心**计算的
- 应用缩放时**直接计算**新的 `_translateX/Y`，不使用增量

#### 当前项目 (ImageViewerControl.xaml.cs) 的坐标系统

```csharp
// 1. ScreenToImageCoord 方法（有问题）
private (double imgX, double imgY) ScreenToImageCoord(double screenX, double screenY)
{
    var containerCenterX = ImageContainer.ActualWidth / 2;
    var containerCenterY = ImageContainer.ActualHeight / 2;

    double imgCenterX = containerCenterX + _translateX;  // ❌ 错误！
    double imgCenterY = containerCenterY + _translateY;  // ❌ 错误！

    double dx = screenX - imgCenterX;
    double dy = screenY - imgCenterY;

    double imgX = dx / _zoomScale;
    double imgY = dy / _zoomScale;

    return (imgX, imgY);
}

// 2. 保存缩放锚点
_zoomAnchorScreenX = pointer.Position.X;  // ❌ 保存的是绝对坐标
_zoomAnchorScreenY = pointer.Position.Y;

// 3. 应用缩放时使用增量方式
if (_hasZoomAnchor)
{
    double imgCenterX = containerCenterX + _translateX;  // ❌ 错误！
    double imgCenterY = containerCenterY + _translateY;

    double dx = _zoomAnchorScreenX - imgCenterX;
    double dy = _zoomAnchorScreenY - imgCenterY;

    double scaleRatio = _zoomScale / oldZoomScale;
    _translateX += dx * (scaleRatio - 1);  // ❌ 增量方式可能累积误差
    _translateY += dy * (scaleRatio - 1);
}
```

**问题**：
1. `imgCenterX = containerCenterX + _translateX` 是错误的计算
   - 图片使用 `HorizontalAlignment="Center"`，图片中心始终在容器中心
   - `_translateX` 是平移变换，不是图片中心的位置

2. `_zoomAnchorScreenX/Y` 保存的是绝对坐标，应该保存相对于容器中心的坐标

3. 使用增量方式计算位移，可能累积误差

## 修复方案

### 方案：采用参考项目的坐标系统

#### 步骤 1：修改滚轮事件中的锚点计算

```csharp
private void ImageContainer_PointerWheelChanged(object sender, PointerRoutedEventArgs e)
{
    var pointer = e.GetCurrentPoint(ImageContainer);
    var delta = pointer.Properties.MouseWheelDelta;
    
    // 计算鼠标相对于容器中心的位置
    var containerCenterX = ImageContainer.ActualWidth / 2;
    var containerCenterY = ImageContainer.ActualHeight / 2;
    var mouseRelativeToCenterX = pointer.Position.X - containerCenterX;
    var mouseRelativeToCenterY = pointer.Position.Y - containerCenterY;
    
    // 计算缩放锚点（图片坐标系，相对于图片中心）
    _zoomAnchorImgX = (mouseRelativeToCenterX - _translateX) / _zoomScale;
    _zoomAnchorImgY = (mouseRelativeToCenterY - _translateY) / _zoomScale;
    
    // 保存相对于容器中心的坐标
    _zoomAnchorScreenX = mouseRelativeToCenterX;
    _zoomAnchorScreenY = mouseRelativeToCenterY;
    _hasZoomAnchor = true;

    // ... 缩放计算 ...
}
```

#### 步骤 2：修改物理引擎中的缩放应用

```csharp
private void OnPhysicsRendering(object? sender, object e)
{
    // ...
    if (isZooming)
    {
        _zoomScale += (_targetZoomScale - _zoomScale) * ZoomEasingFactor;

        if (_hasZoomAnchor)
        {
            // 直接计算新的位移，不使用增量
            _translateX = _zoomAnchorScreenX - _zoomAnchorImgX * _zoomScale;
            _translateY = _zoomAnchorScreenY - _zoomAnchorImgY * _zoomScale;
        }

        ClampTranslation();
        needsUpdate = true;
    }
    // ...
}
```

#### 步骤 3：删除不再需要的 ScreenToImageCoord 方法

该方法可以保留用于其他用途，但缩放锚点计算不再使用它。

## 数学原理

### 缩放锚点公式推导

设：
- `M` = 鼠标相对于容器中心的位置
- `T` = 图片的平移量（相对于容器中心）
- `S` = 缩放比例
- `P` = 鼠标指向的图片上的点（相对于图片中心）

关系：
```
M = T + P * S
```

因此：
```
P = (M - T) / S
```

缩放后，为了保持鼠标指向同一个图片点：
```
M = T' + P * S'
```

所以：
```
T' = M - P * S'
T' = M - ((M - T) / S) * S'
```

这就是参考项目使用的公式：
```csharp
_translateX = _zoomAnchorScreenX - _zoomAnchorImgX * _zoomScale;
```

其中：
- `_zoomAnchorScreenX` = `M`（鼠标相对于容器中心的位置）
- `_zoomAnchorImgX` = `P`（鼠标指向的图片点，相对于图片中心）

## 其他差异

### 1. 100% 吸附逻辑

参考项目有更完善的 100% 吸附逻辑，包括：
- `_justSnappedTo100Percent` 标志
- `_snapStayCounter` 计数器
- 需要滚动多次才能离开 100%

当前项目的吸附逻辑较简单。

### 2. DPI 缩放

参考项目考虑了 `XamlRoot.RasterizationScale`：
```csharp
var dpiScale = XamlRoot?.RasterizationScale ?? 1.0;
var originalScaleForFit = (1.0 / fitScale) / dpiScale;
```

当前项目未考虑 DPI 缩放。

### 3. 旋转支持

参考项目支持旋转和翻转，当前项目不支持。

3. **测试验证** - 确保缩放时鼠标位置与图片点保持对齐
