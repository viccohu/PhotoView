# 分批显示优化方案

## 当前问题分析

### 当前流程（慢）
```
1. 全量扫描所有文件
2. 全部分组
3. 全量加载主图片
4. 最后一次性显示
```
→ 首屏等待时间：几秒到几十秒（取决于文件数）

### 优化建议流程（快）
```
1. 清空界面 → 立刻响应
2. 小批量读取文件（30个/批）
3. 每读完一批 → 立刻分组
4. 每批只加载【主文件】信息
5. 每批立刻追加显示到界面
6. 后台继续跑，不阻塞UI
```
→ 首屏显示：< 300ms

## 实现任务

### [ ] 任务1：修改 LoadImagesAsync - 分批显示
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 批次大小：30（从200改小）
  - 每批处理完立即分组并显示
  - 不等待全部扫描完
  - 分组字典跨批次保留
- **成功标准**：
  - 首屏显示 < 1秒
  - 每批追加显示不卡顿
- **实现要点**：
  ```csharp
  var fileNameMap = new Dictionary<string, List<StorageFile>>(StringComparer.OrdinalIgnoreCase);
  
  while (!cancellationToken.IsCancellationRequested)
  {
      var batch = await result.GetFilesAsync(index, batchSize);
      if (batch.Count == 0) break;
      
      // 1. 加入字典
      foreach (var file in batch)
      {
          var groupName = ImageGroup.GetGroupName(file.Name);
          if (!fileNameMap.TryGetValue(groupName, out var list))
          {
              list = new List<StorageFile>();
              fileNameMap[groupName] = list;
          }
          list.Add(file);
      }
      
      // 2. 处理这批新增的组，只加载主图片
      // ...
      
      index += (uint)batch.Count;
  }
  ```

## 关键优化点

| 优化项 | 说明 |
|--------|------|
| **批次大小** | 30 个/批（快速首屏） |
| **跨批次分组** | fileNameMap 在循环外保留 |
| **每批立刻显示** | 不等待全部扫描完 |
| **只加载主图片** | 副图只存引用 |
