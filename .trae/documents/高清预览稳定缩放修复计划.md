# 高清预览稳定居中自适应缩放修复计划

## 问题分析

### 当前问题
1. **左上角跳动** - `HorizontalAlignment="Center"` 在 Source 切换时重新计算中心点
2. **高度没限制** - 缩放计算未同时约束宽和高
3. **缩略图变小** - 使用原图尺寸数据而非当前 Source 的实际像素尺寸
4. **切换闪烁** - 换图后未重新校准缩放

### 根本原因
- `Stretch="Uniform"` 和 `ZoomFactor` 产生双重叠加
- 使用 `_imageFileInfo` 的原始尺寸而非当前图片的实际像素尺寸
- 高清图替换后未重新计算缩放

---

## 解决方案

### 1. XAML 修改：解除布局竞争

**修改前**：
```xml
<Image Stretch="Uniform" HorizontalAlignment="Center" VerticalAlignment="Center"/>
```

**修改后**：
```xml
<Image Stretch="None" HorizontalAlignment="Left" VerticalAlignment="Top"/>
```

**原理**：
- `Stretch="None"` 让 Image 的物理尺寸等于图片像素尺寸，避免自动拉伸与 ZoomFactor 冲突
- `HorizontalAlignment="Left"` + `VerticalAlignment="Top"` 锁定坐标原点，消除跳动

### 2. 重构缩放算法：基于当前 Source 计算

**修改前**：使用 `_imageFileInfo.Width/Height`
**修改后**：使用 `mainImage.Source` 的 `PixelWidth/PixelHeight`

```csharp
private void CalculateFitScale()
{
    // 核心：获取当前控件中图片的真实像素尺寸
    if (mainImage.Source is not BitmapSource currentSource) return;

    ImageScrollView.UpdateLayout();
    double vWidth = ImageScrollView.ViewportWidth;
    double vHeight = ImageScrollView.ViewportHeight;

    if (vWidth <= 0 || vHeight <= 0) return;

    double iWidth = currentSource.PixelWidth;
    double iHeight = currentSource.PixelHeight;

    // 同时约束宽和高，取最小值实现完整自适应
    double scaleX = vWidth / iWidth;
    double scaleY = vHeight / iHeight;
    _fitScale = Math.Min(scaleX, scaleY);

    // 立即应用缩放，不使用动画以防视觉闪烁
    ImageScrollView.ZoomTo((float)_fitScale, null);
    _hasCalculatedFitScale = true;
}
```

### 3. 优化图片切换流程：两次校准

**阶段一：显示缩略图**
- 在 `PrepareForAnimationAsync` 中，Viewport 准备就绪后，针对缩略图计算一次缩放

**阶段二：替换高清图**
- 在 `LoadHighResolutionImageAsync` 的回调中，替换 Source 后立即再次调用 `CalculateFitScale()`

```csharp
// 在 LoadHighResolutionImageAsync 内部
DispatcherQueue.TryEnqueue(() =>
{
    mainImage.Source = imageSource;  // 换上更高清的图
    mainImage.UpdateLayout();        // 强制测量新图尺寸
    CalculateFitScale();             // 针对新图尺寸重新对齐缩放比
});
```

### 4. 逻辑生命周期

1. **`PrepareContent`**：仅设置 Source 为缩略图并填充文字信息
2. **`PrepareForAnimationAsync`**：
   - 通过循环 `UpdateLayout()` 确保 `ViewportWidth > 0`
   - **执行 `CalculateFitScale()`**：缩略图瞬间对齐视窗
   - 开始加载高清图
3. **`ShowAfterAnimationAsync`**：执行 Opacity 渐变入场动画
4. **高清图加载完成**：自动触发第二次 `CalculateFitScale()`，图片变清晰但位置和大小不变

## 具体修改步骤

### 步骤 1：修改 ImageViewerControl.xaml
- [ ] 将 `Stretch` 改为 `None`
- [ ] 将 `HorizontalAlignment` 改为 `Left`
- [ ] 将 `VerticalAlignment` 改为 `Top`

### 步骤 2：修改 ImageViewerControl.xaml.cs
- [ ] 重构 `CalculateFitScale()` 方法，使用当前 Source 的像素尺寸
- [ ] 在 `CalculateFitScale()` 中调用 `ZoomTo()` 应用缩放
- [ ] 修改 `LoadHighResolutionImageAsync()`，替换高清图后调用 `CalculateFitScale()`
- [ ] 移除 `_pendingHighResImageSource` 机制（不再需要）
- [ ] 简化 `ShowAfterAnimationAsync()`，移除高清图替换逻辑

## 预期效果

| 现象 | 修复原理 |
|-----|---------|
| 左上角跳动 | `HorizontalAlignment="Left"` 锁死物理原点 |
| 高度没限制 | `Math.Min(scaleX, scaleY)` 强制宽高都纳入缩放权重 |
| 缩略图变小 | 放弃原图尺寸数据，改用 `currentSource.PixelWidth` 实时计算 |
