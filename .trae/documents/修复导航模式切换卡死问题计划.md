# 修复导航模式切换卡死问题计划

## 问题分析

### 根本原因
在 `SettingsService.cs` 中，`SaveNavigationViewMode` 和 `LoadNavigationViewMode` 方法使用了 `.Wait()` 和 `.Result` 同步阻塞异步操作，导致死锁：

```csharp
// 问题代码
public void SaveNavigationViewMode(NavigationViewPaneDisplayMode mode)
{
    _localSettingsService.SaveSettingAsync("NavigationViewMode", mode.ToString()).Wait(); // 死锁！
}

public NavigationViewPaneDisplayMode LoadNavigationViewMode()
{
    var mode = _localSettingsService.ReadSettingAsync<string>("NavigationViewMode").Result; // 死锁！
    ...
}
```

### 死锁原理
1. UI 线程调用 `SaveNavigationViewMode`
2. `.Wait()` 阻塞 UI 线程
3. 异步操作需要回到 UI 线程完成（上下文捕获）
4. UI 线程被阻塞，无法处理回调
5. 死锁！

### 官方实现对比
官方使用 `NavigationOrientationHelper.IsLeftModeForElement` 直接同步设置导航模式，不涉及异步存储操作。

## [ ] 任务 1：修复 SettingsService 异步死锁问题
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 将 `SaveNavigationViewMode` 改为异步方法 `SaveNavigationViewModeAsync`
  - 将 `LoadNavigationViewMode` 改为异步方法 `LoadNavigationViewModeAsync`
  - 使用 `ConfigureAwait(false)` 避免上下文捕获
  - 更新接口定义
- **成功标准**：
  - 异步方法正确实现
  - 不再使用 `.Wait()` 或 `.Result`
- **测试要求**：
  - `programmatic` TR-1.1：代码编译无错误
  - `programmatic` TR-1.2：不再有同步阻塞异步操作
- **注意**：需要同时修改接口和实现
