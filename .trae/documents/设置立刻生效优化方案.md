# 设置立刻生效优化方案

## 问题分析

当前实现：

| 设置 | 立刻生效 | 保存 | 说明 |
|------|----------|------|------|
| 主题 | ✅ | ✅ | SetThemeAsync 立刻更新 UI |
| 导航模式 | ✅ | ✅ | NavigationViewModeChanged 事件触发 ShellPage 更新 |
| BatchSize | ❌ | ✅ | 只保存，没有立刻生效机制 |

## 问题原因

BatchSize 设置后：
1. ✅ 保存到本地存储
2. ✅ 更新 SettingsService._batchSize
3. ❌ 没有通知 MainViewModel
4. ❌ 当前文件夹不会重新加载

## 解决方案

### 方案 A：订阅 BatchSizeChanged 事件（推荐）

让 MainViewModel 订阅 `BatchSizeChanged` 事件，设置改变时重新加载当前文件夹。

### 方案 B：不重新加载，只在下次加载时生效

当前实现已经满足这个需求，但用户可能期望立刻看到效果。
