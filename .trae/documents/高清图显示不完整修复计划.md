# 高清图显示不完整问题修复计划

## 问题分析

### 从日志中发现的问题

1. **DisplayInformation 错误**
   ```
   Windows.Graphics.Display: GetForCurrentView must be called on a thread that is associated with a CoreWindow.
   ```
   - `GetForCurrentView()` 必须在 UI 线程调用，但在 `LoadHighResolutionImageAsync`（后台线程）中被调用
   - 导致使用默认的 1920 解码尺寸

2. **高清图尺寸获取失败**
   ```
   [ImageViewer] LoadHighResolutionImageAsync: 无法获取图片尺寸，保持原缩放: 2.578
   ```
   - 高清图替换后无法获取其尺寸，保持了缩略图的缩放比例

3. **缩放比例问题**
   - 缩略图：256x171，缩放比例 2.578
   - 高清图：6040x4032，使用同样的缩放比例 2.578，会导致图片过大

### 根本原因
- 解码尺寸计算在后台线程，无法获取显示器分辨率
- 高清图尺寸获取失败，导致使用错误的缩放比例

---

## 解决方案

### 1. 移到 UI 线程获取显示器分辨率
- 在 `PrepareContent` 或 `PrepareForAnimationAsync` 中获取显示器分辨率
- 缓存到字段中供后台线程使用

### 2. 修复高清图尺寸获取
- 确保 `imageSource` 是 `BitmapSource` 类型
- 添加错误处理和重试机制

### 3. 优化缩放计算
- 确保高清图替换后重新计算正确的缩放比例
- 处理尺寸获取失败的情况

---

## 修改步骤

### 步骤 1：添加分辨率缓存字段
```csharp
private uint _targetDecodeLongSide = 1920;
```

### 步骤 2：在 PrepareForAnimationAsync 中获取分辨率
```csharp
public async Task PrepareForAnimationAsync()
{
    // 先获取显示器分辨率
    _targetDecodeLongSide = GetTargetDecodeLongSide();
    
    // 其他代码...
}
```

### 步骤 3：修改 LoadHighResolutionImageAsync 使用缓存的分辨率
```csharp
var targetLongSide = _targetDecodeLongSide;
```

### 步骤 4：修复高清图尺寸获取
```csharp
if (imageSource is BitmapSource newSource)
{
    // 尝试多次获取尺寸
    int retry = 0;
    while (retry < 3)
    {
        try
        {
            double iWidth = newSource.PixelWidth;
            double iHeight = newSource.PixelHeight;
            if (iWidth > 0 && iHeight > 0)
            {
                // 计算缩放...
                break;
            }
        }
        catch
        {
            await Task.Delay(10);
            retry++;
        }
    }
}
```
