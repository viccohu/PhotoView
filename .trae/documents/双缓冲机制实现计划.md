# LinedFlowLayout 清除布局缓存解决方案

## 问题分析

### 问题描述

切换目录时，LinedFlowLayout 布局混乱：间距爆炸、断层、空一大段、乱换行。

### 根本原因

LinedFlowLayout 内部会缓存行布局信息：

* 每一行的宽度

* 每一项的宽度

* 剩余空间怎么分配

当切换目录时，旧的布局缓存没有被清除，导致新图片使用了旧目录的布局宽度去排版。

## 官方解决方案

### 方法：`InvalidateItemsInfo()`

根据 [Microsoft 官方文档](https://learn.microsoft.com/en-us/windows/windows-app-sdk/api/winrt/microsoft.ui.xaml.controls.linedflowlayout.invalidateitemsinfo)：

```csharp
public void InvalidateItemsInfo();
```

**功能**：

* 让 LinedFlowLayout 重新排列所有项目

* 丢弃之前通过 `ItemsInfoRequested` 事件收集的所有项目尺寸信息

* 触发 `ItemsInfoRequested` 事件来获取最新的项目尺寸信息

### 当前代码状态

**已实现** ([MainPage.xaml.cs:264-275](file:///d:/FastPick/PhotoView/PhotoView/Views/MainPage.xaml.cs#L264-L275))：

```csharp
private void ViewModel_ImagesChanged(object? sender, System.EventArgs e)
{
    SelectionService.ClearSelection();
    ImageScrollView.ScrollTo(0, 0);
    
    ImageLayout.InvalidateItemsInfo();
    
    DispatcherQueue.TryEnqueue(Microsoft.UI.Dispatching.DispatcherQueuePriority.Low, () =>
    {
        ImageLayout.InvalidateItemsInfo();
    });
}
```

## 验证要点

### 1. 确认 `InvalidateItemsInfo()` 调用时机

* ✅ 在 `ImagesChanged` 事件中调用

* ✅ 在清空集合后、添加新项目前调用

### 2. 可能的额外优化

如果当前方案仍然有问题，可以考虑：

#### 方案 A：使用 `InvalidateArrange()`

```csharp
ImageLayout.InvalidateArrange();
ImageLayout.InvalidateItemsInfo();
```

#### 方案 B：重置 ItemsStretch

```csharp
ImageLayout.ItemsStretch = Microsoft.UI.Xaml.Controls.LinedFlowLayoutItemsStretch.None;
ImageLayout.InvalidateItemsInfo();
```

#### 方案 C：延迟重置

```csharp
// 先清空
Images.Clear();

// 延迟加载新数据
DispatcherQueue.TryEnqueue(Microsoft.UI.Dispatching.DispatcherQueuePriority.Low, () =>
{
    // 加载新数据
    LoadNewImages();
    
    // 再次刷新布局
    ImageLayout.InvalidateItemsInfo();
});
```

## 结论

当前代码已经使用了官方推荐的 `InvalidateItemsInfo()` 方法。如果问题仍然存在，需要进一步排查：

1. **检查调用时机** - 确保在正确的时机调用
2. **检查 ItemsStretch 设置** - 当前设置为 `None`，可能需要改为 `Fill`
3. **检查 LineHeight 绑定** - 确保动态绑定正常工作
