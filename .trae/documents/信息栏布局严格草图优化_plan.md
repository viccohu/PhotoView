
# 信息栏布局严格草图优化方案

## 1. 现状分析
当前布局与草图相比存在以下问题：
- 日期时间是整合显示的，没有按草图分成多个方块
- 文件信息和设备信息的方块大小不统一
- 评级控件没有放在按钮样式的容器中

## 2. 优化目标
严格按照草图样式重新布局：
- 所有一级项都是图标+内容的结构
- 信息中的方框统一大小、统一间距
- 日期时间按草图分成多个方块展示
- 文件信息和设备信息的方块严格统一尺寸
- 沿用之前的逻辑展示，不搞乱数据绑定和事件处理

## 3. 具体修改方案

### 3.1 ViewModel 层修改（ImageViewerViewModel.cs）
- 将 FormattedDateTime 拆分为单独的年份、月份、日期、时间属性
- 或者保留 FormattedDateTime 同时添加拆分后的属性

### 3.2 View 层修改（ImageViewerControl.xaml）
1. **文件名区域**：保持图标+内容，但确保样式统一
2. **评级区域**：将评级控件放在统一的按钮样式容器中
3. **日期时间区域**：拆分为年、月、日、时间四个方块
4. **文件信息区域**：使用 UniformGrid 或严格的 Grid 确保所有方块大小一致
5. **设备信息区域**：同样使用严格的网格布局确保方块大小一致

### 3.3 样式调整
- 统一 InfoCardStyle 的 Padding 和最小尺寸
- 确保所有方块有相同的最小宽度和高度
- 统一间距

## 4. 修改文件列表
1. `ViewModels/ImageViewerViewModel.cs` - 添加拆分的日期时间属性
2. `Controls/ImageViewerControl.xaml` - 严格按草图重新布局

## 5. 风险和注意事项
- 保持数据绑定正确
- 保持事件处理逻辑不变
- 使用主题色，不自定义颜色
- 确保响应式布局
