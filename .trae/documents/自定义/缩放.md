资料一：官方应用为了实现那种“丝滑且图片永远居中”的体验，在 `ScrollView` 内部通常会嵌套一个容器（比如 `Grid` 或 `Border`）。

这样做主要解决两个痛点：

1. **居中对齐**：当图片比窗口小时，直接放 `Image` 很难在缩放过程中保持完美的物理居中。
2. **坐标转换**：中间层提供了一个稳定的坐标系，方便处理鼠标拖拽和复杂的缩放偏移计算。

下面是按照“官方风格”封装的**最细致方案**：

### 1. XAML 结构：三层架构

- **外层 (`ScrollView`)**：负责整体的视口控制和滚动条。
- **中层 (`Grid`)**：作为“载体”，其大小会随图片缩放而改变，负责撑开滚动区域。
- **内层 (`Image`)**：实际显示内容，设置 `Stretch="Uniform"`。

XML

```
<ScrollView x:Name="MainScrollView"
            HorizontalScrollBarVisibility="Auto"
            VerticalScrollBarVisibility="Auto"
            ZoomMode="Enabled" 
            MinZoomFactor="0.1" 
            MaxZoomFactor="5.0"
            Background="{ThemeResource ApplicationPageBackgroundThemeBrush}">
    
    <Grid x:Name="ContentContainer"
          HorizontalAlignment="Center" 
          VerticalAlignment="Center"
          PointerWheelChanged="OnPointerWheelChanged"
          PointerPressed="OnPointerPressed"
          PointerMoved="OnPointerMoved"
          PointerReleased="OnPointerReleased"
          PointerCanceled="OnPointerReleased">
        
        <Image x:Name="DisplayImage"
               Source="ms-appx:///Assets/YourPhoto.jpg"
               Stretch="None" 
               HorizontalAlignment="Center" 
               VerticalAlignment="Center"/>
    </Grid>
</ScrollView>

```

### 2. 后端代码：实现“以鼠标为中心”缩放

官方看图最核心的体验是：**你鼠标指着哪个像素，缩放时那个像素就固定在鼠标下方。**

C#

```
private void OnPointerWheelChanged(object sender, PointerRoutedEventArgs e)
{
    var pointerPoint = e.GetCurrentPoint(ContentContainer);
    int delta = pointerPoint.Properties.MouseWheelDelta;
    
    // 获取当前缩放和偏移
    double oldZoom = MainScrollView.ZoomFactor;
    // 获取鼠标相对于 ScrollView 视口的位置
    var viewPoint = e.GetCurrentPoint(MainScrollView).Position;

    // 计算新缩放比例
    double zoomChange = delta > 0 ? 1.1 : 0.9;
    float newZoom = (float)Math.Clamp(oldZoom * zoomChange, MainScrollView.MinZoomFactor, MainScrollView.MaxZoomFactor);

    if (newZoom != oldZoom)
    {
        double ratio = newZoom / oldZoom;

        // 核心公式：计算新的滚动偏移量，确保鼠标指向的坐标不变
        double targetH = (MainScrollView.HorizontalOffset + viewPoint.X) * ratio - viewPoint.X;
        double targetV = (MainScrollView.VerticalOffset + viewPoint.Y) * ratio - viewPoint.Y;

        // 使用 Disabled 模式获得最即时的跟手感，或者使用 Enabled 获得丝滑感
        MainScrollView.ScrollTo(targetH, targetV, new ScrollingScrollOptions(ScrollingAnimationMode.Disabled));
        MainScrollView.ZoomTo(newZoom, new ScrollingZoomOptions(ScrollingAnimationMode.Disabled));
    }

    e.Handled = true;
}

```

### 3. 后端代码：实现左键平滑拖拽

拖拽时，我们需要锁定鼠标，并实时计算位移。

C#

```
private Point _lastMousePosition;
private bool _isDragging = false;

private void OnPointerPressed(object sender, PointerRoutedEventArgs e)
{
    var ptr = e.GetCurrentPoint(MainScrollView);
    if (ptr.Properties.IsLeftButtonPressed)
    {
        _isDragging = true;
        _lastMousePosition = ptr.Position;
        ContentContainer.CapturePointer(e.Pointer);
        // 改变光标为抓取手势
        ContentContainer.ProtectedCursor = InputSystemCursor.Create(InputSystemCursorShape.Hand);
    }
}

private void OnPointerMoved(object sender, PointerRoutedEventArgs e)
{
    if (_isDragging)
    {
        var currentPos = e.GetCurrentPoint(MainScrollView).Position;
        
        // 计算鼠标移动的距离
        double deltaX = _lastMousePosition.X - currentPos.X;
        double deltaY = _lastMousePosition.Y - currentPos.Y;

        // 更新滚动位置
        MainScrollView.ScrollTo(
            MainScrollView.HorizontalOffset + deltaX, 
            MainScrollView.VerticalOffset + deltaY, 
            new ScrollingScrollOptions(ScrollingAnimationMode.Disabled));

        _lastMousePosition = currentPos;
    }
}

private void OnPointerReleased(object sender, PointerRoutedEventArgs e)
{
    _isDragging = false;
    ContentContainer.ReleasePointerCapture(e.Pointer);
    ContentContainer.ProtectedCursor = null; // 恢复光标
}

```

### 4. 为什么官方方案要这样写？（细节揭秘）

1. **关于** **`ScrollingAnimationMode`**： 如果你想要完全像 Windows 照片应用那样，滚轮滚动时有一点点“Q弹”的缓冲感，可以把 `Disabled` 改为 `Enabled`。但如果追求极致的精准（比如查看工程图），`Disabled` 更合适。
2. **关于** **`ContentContainer`** **的作用**： 你会发现 `Image` 的 `Stretch` 设为了 `None`。这是因为 `ScrollView` 的缩放本质上是对其 `Content` 进行渲染变换（RenderTransform）。如果 `Image` 随着父容器自动拉伸，会导致缩放算法变得极其复杂。让 `Image` 保持原始像素大小，完全交给 `ScrollView` 去“放大视图”，性能是最好的。
3. **防止闪烁**： 在 WinUI 3 中，`ScrollView` 的 `ScrollTo` 和 `ZoomTo` 如果在同一帧调用，可能会有微小的不同步。官方推荐将它们封装在同一个交互逻辑中处理，以保证视觉连贯性。
4. 如果你追求的是**100% 还原 Windows 11 “照片”应用**那种极致平滑、带有惯性、且缩放中心精准的效果，我必须向你纠正一个细节：
   资料二：
   **官方（Photos App）确实是用了** **`ScrollView`（或旧版的** **`ScrollViewer`），但它并不是简单的“设置属性”，而是利用了 DirectComposition 层面的位图处理。**

   为了让你达到“像素级”的官方体验，这里有三个你必须处理的**深层细节**，这是普通教程不会告诉你的：
   ***
   ### 1. 为什么一定要加那一层容器？（官方的秘密）
   官方在 `ScrollView` 内部套一个 `Grid` 或 `Border`（通常命名为 `Viewport`），是因为：
   - **布局隔离**：`Image` 控件如果直接缩放，它的 `ActualWidth` 会变，这会导致 `ScrollView` 重新计算布局，产生抖动。
   - **背景填充**：官方看图在图片比窗口小时，背景是黑色或主题色；图片比窗口大时，背景被图片填满。这个逻辑必须由中间层 `Grid` 配合 `HorizontalAlignment="Center"` 来实现。
   ***
   ### 2. 官方级的“缩放中心”数学模型
   官方应用在滚轮缩放时，**不是**在 `PointerWheelChanged` 里直接改 `ZoomFactor`。 因为 `ScrollView.ZoomTo` 会触发动画引擎，如果你连续滚动滚轮，动画会堆叠，导致延迟感。

   **官方逻辑：**
   1. 监听滚轮，计算目标 `ZoomFactor`。
   2. 立即计算出在该缩放比例下，鼠标所指像素点对应的 `HorizontalOffset` 和 `VerticalOffset`。
   3. **关键点：** 同时调用 `ScrollTo` 和 `ZoomTo`，并将 `ScrollingAnimationMode` 设为 `Disabled` 以获取即时反馈，或者使用 `Enabled` 但必须处理 `AnchorPoint`（锚点）。
   ***

   // 缩放核心：以鼠标为中心
   private void ImageHost_PointerWheelChanged(object sender, PointerRoutedEventArgs e)
   {
       var pt = e.GetCurrentPoint(MainScrollView);
       double delta = pt.Properties.MouseWheelDelta;
       
       float oldZoom = MainScrollView.ZoomFactor;
       float zoomStep = 1.1f;
       float newZoom = delta > 0 ? oldZoom * zoomStep : oldZoom / zoomStep;
       newZoom = Math.Clamp(newZoom, (float)MainScrollView.MinZoomFactor, (float)MainScrollView.MaxZoomFactor);

       if (newZoom != oldZoom)
       {
           // 计算鼠标在当前视口中的位置比例
           double mouseX = pt.Position.X;
           double mouseY = pt.Position.Y;

           double scrollX = MainScrollView.HorizontalOffset;
           double scrollY = MainScrollView.VerticalOffset;

           double ratio = newZoom / oldZoom;

           // 核心公式：计算新的滚动位置以抵消缩放产生的位移
           double targetX = (scrollX + mouseX) * ratio - mouseX;
           double targetY = (scrollY + mouseY) * ratio - mouseY;

           // 官方级体验：不带延迟的即时响应
           MainScrollView.ScrollTo(targetX, targetY, new ScrollingScrollOptions(ScrollingAnimationMode.Disabled));
           MainScrollView.ZoomTo(newZoom, new ScrollingZoomOptions(ScrollingAnimationMode.Disabled));
       }
       e.Handled = true;
   }

   // 拖拽核心：左键抓取
   private Point _lastPos;
   private bool _isPressed = false;

   private void ImageHost_PointerPressed(object sender, PointerRoutedEventArgs e)
   {
       var ptr = e.GetCurrentPoint(MainScrollView);
       if (ptr.Properties.IsLeftButtonPressed)
       {
           _isPressed = true;
           _lastPos = ptr.Position;
           ImageHost.CapturePointer(e.Pointer);
           // 官方手势：抓取
           ImageHost.ProtectedCursor = InputSystemCursor.Create(InputSystemCursorShape.Hand);
       }
   }

   private void ImageHost_PointerMoved(object sender, PointerRoutedEventArgs e)
   {
       if (_isPressed)
       {
           var currentPos = e.GetCurrentPoint(MainScrollView).Position;
           
           // 计算偏移并直接滚动
           double dX = _lastPos.X - currentPos.X;
           double dY = _lastPos.Y - currentPos.Y;

           MainScrollView.ScrollTo(
               MainScrollView.HorizontalOffset + dX, 
               MainScrollView.VerticalOffset + dY, 
               new ScrollingScrollOptions(ScrollingAnimationMode.Disabled));

           _lastPos = currentPos;
       }
   }

   private void ImageHost_PointerReleased(object sender, PointerRoutedEventArgs e)
   {
       _isPressed = false;
       ImageHost.ReleasePointerCapture(e.Pointer);
       ImageHost.ProtectedCursor = null;

   ### 4. 终极差异：为什么官方看起来更“丝滑”？
   如果你觉得代码写完后还是不如官方顺滑，那是因为官方应用还做了以下两点：
   1. **硬件加速解码**：官方使用 `BitmapImage` 的 `DecodePixelWidth` 动态调整解码大小，防止大图占用过多显存。
   2. **ManipulationMode**：如果你想要支持触控板的捏合手势，你需要设置 `ManipulationMode="All"` 并处理 `ManipulationDelta` 事件。上述代码主要针对**鼠标**用户优化。

资料三：
要实现像官方看图应用那样“直接滚动滚轮即缩放”，核心做法是：**拦截滚轮事件（PointerWheelChanged），通过代码阻止默认滚动，并手动调用缩放 API。**

以下是实现这一行为的三个关键步骤：

### 1. 拦截并标记“已处理”

你必须在事件处理函数的最后加上 `e.Handled = true;`。这会告诉系统：“我已经处理了这个滚轮信号，你不要再执行默认的垂直滚动了。”

### 2. 区分滚轮方向

滚轮向上（`Delta > 0`）对应放大，向下对应缩小。官方应用通常会给一个固定的缩放系数（如 1.1 倍或 0.9 倍）。

### 3. 精准的中心点补偿

这是“官方感”的来源。如果不做补偿，缩放会向左上角偏移。我们需要计算鼠标当前在 `ScrollView` 视口里的位置，并在缩放的同时，通过 `ScrollTo` 把那个点“拽”回鼠标下方。

***

### 完整实现代码（基于 ScrollView）

C#

```
private void ImageHost_PointerWheelChanged(object sender, PointerRoutedEventArgs e)
{
    // 1. 获取鼠标相对于 ScrollView 视口的位置（不是相对于图片的）
    var pointerPoint = e.GetCurrentPoint(MainScrollView);
    double mouseX = pointerPoint.Position.X;
    double mouseY = pointerPoint.Position.Y;

    // 2. 获取当前的滚动偏移和缩放因子
    double oldZoom = MainScrollView.ZoomFactor;
    double scrollX = MainScrollView.HorizontalOffset;
    double scrollY = MainScrollView.VerticalOffset;

    // 3. 计算新的缩放比例
    int delta = pointerPoint.Properties.MouseWheelDelta;
    float zoomFactor = delta > 0 ? 1.1f : 0.9f; // 向上拨动放大10%，向下缩小10%
    float newZoom = (float)Math.Clamp(oldZoom * zoomFactor, MainScrollView.MinZoomFactor, MainScrollView.MaxZoomFactor);

    if (newZoom != oldZoom)
    {
        // 4. 计算偏移补偿量（核心数学公式）
        // 原理：(旧偏移 + 鼠标位置) * 缩放比 - 鼠标位置 = 新偏移
        double ratio = newZoom / oldZoom;
        double targetX = (scrollX + mouseX) * ratio - mouseX;
        double targetY = (scrollY + mouseY) * ratio - mouseY;

        // 5. 立即更新视图
        // 官方做法通常禁用动画 (Disabled)，以获得极其灵敏的跟手感
        MainScrollView.ScrollTo(targetX, targetY, new ScrollingScrollOptions(ScrollingAnimationMode.Disabled));
        MainScrollView.ZoomTo(newZoom, new ScrollingZoomOptions(ScrollingAnimationMode.Disabled));
    }

    // 6. 关键：阻止系统默认的垂直滚动行为
    e.Handled = true;
}

```

***

### 为什么这是“官方”逻辑？

- **打破默认行为**：Windows 系统的辅助功能规定滚轮是滚动，但看图软件属于特殊工作流。通过 `e.Handled = true`，你实质上是接管了输入流。
- **非线性缩放**：官方应用在连续快速滚动时，缩放感非常平滑。如果你想要更高级的体验，可以引入一个 `double _targetZoom` 变量，配合 `Composition` 动画引擎来做平滑过渡，而不是直接硬切坐标。
- **单指/双指区分**：如果你在笔记本触控板上双指滑动，系统也会触发滚轮事件。上述代码同样适用于触控板的双指缩放模拟。

### 避坑指南：

- **不要在** **`Image`** **上监听**：一定要在 `ScrollView` 或者它直接嵌套的容器上监听滚轮。如果在 `Image` 上监听，当图片缩小到比视口小时，鼠标悬停在空白处滚轮就失效了。
- **检查** **`ZoomMode`**：确保 XAML 中 `ZoomMode="Enabled"`，否则 `ZoomTo` 方法可能无效。
