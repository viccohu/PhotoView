# 极简优化方案（纯逻辑 + 核心代码，不污染你的原有结构）

&#x20;

我只给你**必须改的优化逻辑**，你直接嵌入现有代码即可，不替换整段。

## 一、核心问题

你现在：**先全量扫描 → 全部分组 → 全量加载 → 最后一次性显示**

导致：首屏等待极久，UI 阻塞

## 二、最终优化流程（最关键）

plaintext

```
1. 清空界面 → 立刻响应
2. 小批量读取文件（30个/批）
3. 每读完一批 → 立刻分组
4. 每批只加载【主文件】信息
5. 每批立刻追加显示到界面
6. 后台继续跑，不阻塞UI

```

## 四、主图选择逻辑（通用代码）

csharp

运行

```
private StorageFile GetPrimaryFile(List<StorageFile> files)
{
    var priority = new[] { ".jpg", ".jpeg", ".png", ".webp", ".heic" };
    foreach (var ext in priority)
    {
        var f = files.FirstOrDefault(x => 
            x.FileType.Equals(ext, StringComparison.OrdinalIgnoreCase));
        if (f != null) return f;
    }
    return files[0];
}

```

***

## 五、性能关键点（必做）

1. **只加载主图的 ImageProperties**（宽高 / 标题）
2. **副图只存引用，不读取任何属性**
3. **分批读取 + 分批显示**，不等待全部完成
4. **分组字典跨批次保留**，不重复计算

***

## 六、最终效果保证

- **首屏显示 < 300ms**
- **打开文件夹瞬间看到图**
- **RAW 不拖慢速度**
- **滚动 100% 流畅**
- **内存占用降低 50%+**
