# 🚨 一、可能导致崩溃的点（重点）

## ❌ 1. `BitmapImage.SetSource()` 线程问题（高概率崩溃）

```
var bitmapImage = new BitmapImage();
bitmapImage.SetSource(thumbnail);
```

以及：

```
bitmapImage.SetSource(fileStream);
```

***

## 🚨问题

👉 `BitmapImage` 是 **UI 线程对象**

但你现在：

```
GetThumbnailAsync() → 在线程池运行
```

👉 结果：

> ❗ 在非 UI 线程创建/操作 UI 对象 → **直接崩溃（0xc000027b）**

***

## ✅ 正确做法（必须改）

### 🔥 改成 UI线程创建 BitmapImage

```
private async Task<BitmapImage> CreateBitmapAsync(IRandomAccessStream stream)
{
    BitmapImage bitmap = null;

    await App.MainWindow.DispatcherQueue.EnqueueAsync(() =>
    {
        bitmap = new BitmapImage();
        bitmap.SetSource(stream);
    });

    return bitmap!;
}
```

***

然后改：

```
var bitmapImage = await CreateBitmapAsync(thumbnail);
```

***

## ❗这个是你当前最可能的崩溃点之一

***

# ❌ 2. `EnsureThumbnailAsync` 没有“防复用”（会错位/潜在崩）

```
Thumbnail = await GetThumbnailAsync(size);
```

***

## 🚨问题

如果：

1. &#x20;Item A 开始加载&#x20;
2. &#x20;滚动 → A 被复用&#x20;
3. &#x20;await 返回&#x20;
4. &#x20;❌ 设置到了错误的 DataContext&#x20;

***

## ✅ 解决（必须）

👉 加一个“请求版本号”

```
private int _loadVersion;
```

***

```
public async Task EnsureThumbnailAsync(ThumbnailSize size)
{
    if (Thumbnail != null || _isLoading)
        return;

    _isLoading = true;
    var version = ++_loadVersion;

    try
    {
        var result = await GetThumbnailAsync(size);

        // ⚠️ 防止复用错位
        if (version != _loadVersion)
            return;

        Thumbnail = result;
    }
    finally
    {
        _isLoading = false;
    }
}
```

***

# ⚠️ 二、会导致卡顿/性能问题的点

## ⚠️ 3. `_thumbnailCache` 无限制增长

```
private readonly Dictionary<ThumbnailSize, BitmapImage> _thumbnailCache = new();
```

***

## 🚨问题

- &#x20;每张图片缓存多个尺寸&#x20;
- &#x20;不会释放&#x20;
- &#x20;图片多了 → 内存爆炸&#x20;

***

## ✅ 建议

👉 限制只缓存当前尺寸：

```
_thumbnailCache.Clear();
_thumbnailCache[size] = bitmapImage;
```

***

或者：

👉 只缓存一个：

```
private BitmapImage? _thumbnail;
```

（简单且够用）

***

## ⚠️ 4. fallback 加载原图（危险）

```
var fallbackImage = await GetImageSourceAsync();
```

***

## 🚨问题

👉 原图可能是：

```
4000x3000
8000x6000
```

👉 直接：

> ❗ 内存暴涨 + UI卡顿 + 崩溃

***

## ✅ 建议（非常重要）

👉 不要 fallback 原图

```
return new BitmapImage(new Uri("ms-appx:///Assets/placeholder.png"));
```

## ⚠️ 6. AspectRatio +1 hack


## 🚨问题

- &#x20;会影响布局精度&#x20;
