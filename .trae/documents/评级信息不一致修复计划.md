# 评级信息不一致问题修复计划

## 问题分析

### 根本原因
- **ImageFileInfo.Rating**: 0-99 的原始评级值
- **ImageFileInfo.RatingValue**: 转换为星星数 (-1 = 未设置, 0-5 = 星数)
- **缩略图列表**: 使用 `ImageFileInfo.RatingValue` ✅
- **高清预览**: 直接使用 `ImageFileInfo.Rating` (0-99) ❌

### 问题代码位置
`ImageViewerViewModel.cs:119`
```csharp
Rating = imageFileInfo.Rating;  // 直接赋值 0-99，但 RatingControl 期望 0-5
```

`ImageViewerViewModel.cs:431`
```csharp
Rating = rating;  // 同样的问题
```

---

## 修复方案

### 修改 ImageViewerViewModel

**需要添加两个转换方法：**
1. `RatingToStars(uint rating)` - 把 0-99 转换为 0-5 星
2. `StarsToRating(int stars)` - 把 0-5 星转换为 0-99

**修改内容：**
1. 在 `SetBasicInfo` 中，把 `imageFileInfo.Rating` 转换为星星数再赋值给 `Rating`
2. 在 `SetRatingAsync` 中，把用户选择的星星数转换回 0-99 再保存

---

| 文件 | 修改内容 |
|-----|---------|
| `ImageViewerViewModel.cs` | 添加 RatingToStars/StarsToRating 方法，修改 SetBasicInfo 和 SetRatingAsync |

## 修复步骤

1. 添加静态转换方法（复用 ImageFileInfo 的逻辑）
2. 修改 SetBasicInfo：`Rating = RatingToStars(imageFileInfo.Rating)`
3. 修改 SetRatingAsync：保存时用 `StarsToRating(rating)` 转换
