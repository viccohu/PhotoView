# 文件评级功能重构计划

## 目标
重构文件评级读写功能，使用 WinRT API 实现异步评级读写，并在缩略图下方添加评级交互控件。对于不支持 WinRT 评级的文件，使用本地缓存存储评级信息。

## 需求概述
1. 使用 `Windows.Storage.StorageFile.Properties.GetImagePropertiesAsync()` 和 `ImageProperties.Rating` 进行读写
2. 缩略图下方添加 180×35 半透明评级框，内含 RatingControl 控件
3. **所有文件都显示评级框**，未评级显示空星
4. 异步读写时使用 Shimmer 控件展示加载骨架图
5. WinRT 支持的文件格式：JPG/JPEG、TIFF/TIF、PNG（部分）、HEIC/HEIF（Win11+）、DNG
6. **其他文件使用本地缓存存储评级**（文件路径匹配）

### 步骤 4：创建 RatingService 服务
**新文件**: `Services/RatingService.cs`

职责：
- 统一的评级读写接口
- 优先使用 WinRT API，不支持时回退到本地缓存
- 提供文件类型支持检测方法 `IsWinRTRatingSupported(string fileExtension)`
- 使用 `SemaphoreSlim` 限制并发（4-8个）
- 正确处理异常

WinRT 支持的文件扩展名：
```csharp
private static readonly HashSet<string> WinRTSupportedExtensions = new(StringComparer.OrdinalIgnoreCase)
{
    ".jpg", ".jpeg", ".tif", ".tiff", ".png", ".heic", ".heif", ".dng"
};
```

核心逻辑：
```csharp
public async Task<int> GetRatingAsync(StorageFile file)
{
    if (IsWinRTRatingSupported(file.FileType))
    {
        try
        {
            var properties = await file.Properties.GetImagePropertiesAsync();
            return (int)properties.Rating;
        }
        catch
        {
            // WinRT 失败，回退到本地缓存
        }
    }
    
    // 使用本地缓存
    return _cacheService.GetRating(file.Path) ?? 0;
}

public async Task SetRatingAsync(StorageFile file, uint rating)
{
    if (IsWinRTRatingSupported(file.FileType))
    {
        try
        {
            var properties = await file.Properties.GetImagePropertiesAsync();
            properties.Rating = rating;
            await properties.SavePropertiesAsync();
            return;
        }
        catch
        {
            // WinRT 失败，回退到本地缓存
        }
    }
    
    // 使用本地缓存
    await _cacheService.SetRatingAsync(file.Path, (int)rating);
}
```

Rating 值范围：0-99（WinRT API 标准）
- 0 = 未评级
- 1-25 = 1 星
- 26-50 = 2 星
- 51-75 = 3 星
- 76-99 = 4 星（部分系统 76-99 为 5 星）

### 步骤 8：添加必要的转换器
**新文件**: `Helpers/RatingConverter.cs`

将 WinRT Rating 值（0-99）转换为 RatingControl 的星级值（0-5）：
```csharp
public class RatingConverter : IValueConverter
{
    public object Convert(object value, Type targetType, object parameter, string language)
    {
        if (value is int rating)
        {
            // 0-99 转换为 0-5 星
            if (rating == 0) return 0;
            if (rating <= 20) return 1;
            if (rating <= 40) return 2;
            if (rating <= 60) return 3;
            if (rating <= 80) return 4;
            return 5;
        }
        return 0;
    }
    
    public object ConvertBack(object value, Type targetType, object parameter, string language)
    {
        if (value is int stars)
        {
            // 0-5 星转换为 0-99
            return stars switch
            {
                0 => 0,
                1 => 20,
                2 => 40,
                3 => 60,
                4 => 80,
                5 => 99,
                _ => 0
            };
        }
        return 0;
    }
}
```

## 线程安全注意事项

1. **UI 对象必须在 UI 线程创建** - RatingControl 是 UI 控件，已在 XAML 中定义
2. **异步操作后需回到 UI 线程更新属性** - ImageFileInfo 的 OnPropertyChanged 已处理 DispatcherQueue
3. **限制并发** - RatingService 使用 SemaphoreSlim(4) 限制并发文件操作
4. **缓存文件写入锁** - RatingCacheService 使用 SemaphoreSlim(1) 保证写入安全
5. **取消令牌** - 考虑在页面卸载时取消未完成的评级加载

---

## 错误处理

1. WinRT API 不支持 - 回退到本地缓存
2. WinRT API 失败（文件被占用、权限不足） - 回退到本地缓存
3. 缓存文件损坏 - 重新创建空缓存
4. 缓存文件写入失败 - 记录日志，不阻塞用户操作

---

## 评级来源优先级

1. **读取时**: WinRT API → 本地缓存 → 默认值 0
2. **写入时**: WinRT 支持则写入文件属性 → 否则写入本地缓存
